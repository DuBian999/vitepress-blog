# 码制

## 码制类型

| 码制类型 | 核心定义                                                 | 表示规则（以8位二进制为例）                          | 正数示例（+5） | 负数示例（-5） |
| -------- | -------------------------------------------------------- | ---------------------------------------------------- | -------------- | -------------- |
| 原码     | 符号位+数值位，直接表示二进制数，符号位0为正、1为负      | 1位符号位 + 7位数值位，数值位为十进制数的二进制表示  | 00000101B      | 10000101B      |
| 反码     | 基于原码推导，用于过渡到补码，符号位不变，数值位按位取反 | 正数：与原码一致；负数：符号位为1，数值位取反        | 00000101B      | 11111010B      |
| 补码     | 计算机实际存储和运算使用的码制，解决原码减法运算问题     | 正数：与原码一致；负数：反码 + 1（若有进位则舍弃）   | 00000101B      | 11111011B      |
| 移码     | 用于表示浮点数的阶码，基于补码推导，符号位与补码相反     | 8位移码：补码的符号位取反，数值位不变（偏移量为128） | 10000101B      | 01111011B      |

易错提醒：

- 8位二进制原码/反码范围：-127~+127，补码范围：-128~+127；

- 正数的原码、反码、补码完全一致；

- 移码仅用于阶码，符号位与补码相反。

## 正数码值转换规则（原码=反码=补码）

| 转换方向    | 规则描述                             | 示例（真值：+5）                  |
| ----------- | ------------------------------------ | --------------------------------- |
| 原码 → 反码 | 符号位、数值位均不变，与原码完全一致 | 原码：0000 0101 → 反码：0000 0101 |
| 原码 → 补码 | 符号位、数值位均不变，与原码完全一致 | 原码：0000 0101 → 补码：0000 0101 |
| 补码 → 移码 | 数值位保持不变，仅将符号位取反       | 补码：0000 0101 → 移码：1000 0101 |

## 负数码值转换规则

| 转换方向                | 规则描述                                                                               | 示例（真值：-5）                  |
| ----------------------- | -------------------------------------------------------------------------------------- | --------------------------------- |
| 原码 → 反码             | 符号位保持不变，数值位每一位都取反（0变1，1变0）                                       | 原码：1000 0101 → 反码：1111 1010 |
| 反码 → 补码             | 符号位参与运算，在反码的基础上整体加1（若有进位则依次进位）                            | 反码：1111 1010 → 补码：1111 1011 |
| 补码 → 反码             | 方法1：补码整体减1；方法2：符号位不变，数值位从右往左第1个1的左侧全部取反              | 补码：1111 1011 → 反码：1111 1010 |
| 补码 → 原码             | 方法1：对补码再次求补（反码+1）；方法2：符号位不变，数值位从右往左第1个1的左侧全部取反 | 补码：1111 1011 → 原码：1000 0101 |
| 补码 → 移码             | 数值位保持不变，仅将符号位取反                                                         | 补码：1111 1011 → 移码：0111 1011 |
| 原码 → 补码（快捷方式） | 符号位不变，数值位从右往左第1个1的左侧全部取反（无需先求反码）                         | 原码：1000 0101 → 补码：1111 1011 |

## 码制转换示例（8位机器数）

| 真值 | 原码          | 反码          | 补码      | 移码      |
| ---- | ------------- | ------------- | --------- | --------- |
| +5   | 0000 0101     | 0000 0101     | 0000 0101 | 1000 0101 |
| -5   | 1000 0101     | 1111 1010     | 1111 1011 | 0111 1011 |
| +0   | 0000 0000     | 0000 0000     | 0000 0000 | 1000 0000 |
| -0   | 1000 0000     | 1111 1111     | 0000 0000 | 1000 0000 |
| +127 | 0111 1111     | 0111 1111     | 0111 1111 | 1111 1111 |
| -128 | —（无法表示） | —（无法表示） | 1000 0000 | 0000 0000 |

## n位二进制数（定点整数）的数值范围和数码个数

| 定点整数类型 | 数值范围（十进制） | 数码个数 | 软考备注（关键提醒） |
|-------------|-------------------|---------|-------------------|
| 无符号n位二进制定点整数 | 0 ~ 2<sup>n</sup> - 1 | 2<sup>n</sup> 个 | 无符号，仅表示正整数，无符号位，如8位：0~255 |
| 有符号n位二进制定点整数（原码/反码） | -(2<sup>n-1</sup> - 1) ~ 2<sup>n-1</sup> - 1 | 2<sup>n</sup> - 1 个 | 1位符号位（0正1负）+ n-1位数值位，存在+0和-0，如8位：-127~+127 |
| 有符号n位二进制定点整数（补码） | -2<sup>n-1</sup> ~ 2<sup>n-1</sup> - 1 | 2<sup>n</sup> 个 | 计算机实际存储运算用补码，无±0区分，如8位：-128~+127（软考重点） |

## n位二进制数（定点小数）的数值范围和数码个数

| 定点小数类型 | 数值范围（十进制） | 数码个数 | 软考备注（关键提醒） |
|-------------|-------------------|---------|-------------------|
| 无符号n位二进制定点小数 | 0 ~ 1 - 2<sup>-n</sup> | 2<sup>n</sup> 个 | 小数点固定在最高位右侧，n位均为小数位，无符号位，如4位：0~0.9375 |
| 有符号n位二进制定点小数（原码/反码） | -(1 - 2<sup>-(n-1)</sup>) ~ 1 - 2<sup>-(n-1)</sup> | 2<sup>n</sup> - 1 个 | 1位符号位+ n-1位小数位，存在+0和-0，如4位原码：-0.875~+0.875 |
| 有符号n位二进制定点小数（补码） | -1 ~ 1 - 2<sup>-(n-1)</sup> | 2<sup>n</sup> 个 | 1位符号位+ n-1位小数位，无±0，如4位补码：-1~+0.875（软考高频） |
