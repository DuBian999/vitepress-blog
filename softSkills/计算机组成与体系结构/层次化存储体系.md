层次化存储体系的核心目标是平衡访问速度、存储容量与单位成本，采用“金字塔结构”，自顶向下速度递减、容量递增、成本递减

| 存储层级 | 核心特点（软考高频） | 访问速度/容量/成本 | 存取方式（软考高频） | 核心功能/关键细节（软考高频） |
|---------|-------------------|------------------|-------------------|---------------------------|
| 寄存器 | - 位于CPU内核中<br>- 由硬件直接调度，对软件透明<br>- 易失性（断电丢失数据） | - 速度最快（约1ns）<br>- 容量最小（KB级）<br>- 单位成本最高 | - 随机存取（RAM）<br>- 硬件直接读写，无寻址延迟，随取随用 | - 暂存当前运算的指令和数据<br>- 与CPU运算器、控制器直接交互，无延迟<br>- 软考常考：属于CPU内部组成，是存储体系最顶层 |
| Cache（高速缓冲存储器） | - 位于CPU与主存之间<br>- 分L1、L2、L3三级（L1/L2集成于CPU）<br>- 对软件透明，基于局部性原理工作 | - 速度较快（约1-10ns）<br>- 容量较小（MB级）<br>- 单位成本较高 | - 随机存取（RAM）<br>- 写操作方式（软考常考）：写直达、写回、写分配、非写分配 | - 缓存主存中高频访问的内容，缓解CPU与主存速度差<br>- 核心考点：地址映射、命中率计算<br>- 命中率核心公式（软考必考）： H = 命中次数（N<sub>c</sub>）/ 总访问次数（N<sub>c</sub> + 未命中次数（N<sub>m</sub>））<br>- 平均访存时间（软考常考）： 平均访存时间 = H × T<sub>c</sub> + (1-H) × T<sub>m</sub> （T<sub>c</sub> 为Cache访问时间，T<sub>m</sub> 为主存访问时间） |
| 主存（内存） | - 使用DRAM实现，需定期刷新<br>- CPU可直接寻址访问<br>- 易失性（断电丢失数据） | - 速度中等（约10-100ns）<br>- 容量中等（GB级，主流）<br>- 单位成本中等 | - 随机存取（RAM），支持按地址直接读写<br>- 包含只读存取（ROM）：BIOS常用，断电不丢失数据 | - 存储当前运行的程序和数据<br>- 软考常考：DRAM与SRAM对比（SRAM用于Cache，无需刷新）<br>- 刷新方式：集中式、分散式、异步刷新（主流） |
| 辅存（外存） | - 包括硬盘、SSD、光盘、磁带等<br>- 非易失性（断电不丢失数据）<br>- CPU无法直接访问，需通过I/O接口加载至主存 | - 速度最慢（约100ns-10ms）<br>- 容量最大（TB级）<br>- 单位成本最低 | - 分类型存取：硬盘/磁带为顺序存取+直接存取<br>- 光盘为随机存取<br>- 闪存（Flash）：按块擦写，支持随机读取、顺序写入 | - 持久化存储大量程序和数据<br>- 软考常考：虚拟存储体系由主存+辅存构成<br>- 闪存（Flash）特性：按块擦写，有擦写寿命限制 |
